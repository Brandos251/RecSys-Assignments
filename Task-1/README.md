## Цель
Реализовать расчет **косинусного подобия** между пользователями и товарами на основе матрицы предпочтений (MP), чтобы определить наиболее близких пользователей и товары.

## Матрица предпочтений
Матрица представлена в виде списка списков:

|    | u1 | u2 | u3 | u4 | u5 |
|----|----|----|----|----|----|
| p1 | 5  | 4  | 5  | 3  | 5  |
| p2 | 5  | 5  | 5  | 3  | 5  |
| p3 | 5  | 4  | 4  | 2  | 5  |
| p4 | 5  | 3  | 5  | 0  | 3  |
| p5 | 5  | 0  | 5  | 0  | 0  |
| p6 | 4  | 5  | 5  | 3  | 1  |

- Столбцы — пользователи (U1–U5)  
- Строки — продукты (P1–P6)  
- Значения — оценки пользователей для каждого продукта  

## Алгоритм работы
1. **Создание векторов**:  
   - Каждый пользователь и каждый продукт представляется как вектор оценок.  
   - Для пользователей: вектор — это столбец в матрице MP.  
   - Для продуктов: вектор — это строка в матрице MP.  

2. **Расчет косинусного подобия**:  
   - Для каждой пары пользователей и каждой пары продуктов рассчитывается косинусное подобие.  
   - Формула косинусного подобия между векторами A и B:

3. **Поиск наиболее близких объектов**:  
   - Находим максимальное косинусное подобие среди пользователей и продуктов.  
   - Пользователи и продукты с максимальным значением считаются наиболее близкими.

## Результат
Программа выводит:

1. Матрицу предпочтений.  
2. Общие косинусные подобия по пользователям и по продуктам.  
3. Наиболее близких пользователей и товаров со значением косинусного подобия.

Пример вывода:

Наиболее близкие пользователи: U1 и U2 со значением подобия = 0.987

Наиболее близкие продукты: P1 и P2 со значением подобия = 0.992
