## Цель
Реализовать **четыре варианта хранения матрицы предпочтений** для рекомендательной системы:

1. **Координатное хранение** – хранение ненулевых значений с указанием их позиций (строка, столбец).  
2. **Сжатый строчный формат (Compressed Row Storage, CSR)** – хранение ненулевых значений, столбцов и индексов строк для быстрого доступа.  
3. **Ellpack** – выравнивание строк, добавление нулей для формирования равных длин векторов.  
4. **Сокращение матрицы** – удаление товаров с нулевым или низким рейтингом, а также неактивных пользователей.
